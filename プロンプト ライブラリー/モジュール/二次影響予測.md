#### ♟️ Module Q-2: 二次的思考 (Second-Order Thinking)

**【機能】** 「その解決策を実行したら、次に何が起きるか？」という**ドミノ倒しの2枚目以降**を予測させます。

**【用途】** 提案が「良さそう」に見える時こそ使うべきです。副作用や、成功した後に来る新たな問題を予見します。

  

```xml

<instruction>

  提案されている解決策に対し、「二次的思考（Second-Order Thinking）」による影響評価を行ってください。

  <levels>

    *   **First Order (直接的結果):** 提案を実行してすぐに得られるメリットは何か？（現在の議論）

    *   **Second Order (二次的結果):** そのメリットが得られた結果、次に何が起こるか？（副作用、市場の反応、競合の動き）

    *   **Third Order (長期的帰結):** それが繰り返された1年後、システムはどう変化しているか？

  </levels>

  <output_requirement>

    二次・三次影響において「ネガティブな連鎖」が見つかった場合、現在の提案にどのような修正を加えるべきか提示せよ。

  </output_requirement>

</instruction>
