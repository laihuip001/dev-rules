# Antigravity Optimization Protocol (v1.0)

Derived from the Prompt Library Optimization project.

## 1. Structural Unification (構造的統合)

- **Concept:** audit (監査) and fix (修正) are inseparable twins.
- **Rule:** Use `modes: [audit, fix]` in YAML frontmatter. combine paired concepts into a single physical file to reduce context switching.

## 2. Aesthetic Flattening (XML排除と平坦化)

- **Concept:** Deep XML nesting reduces readability and wastes tokens.
- **Rule:** Replace XML tags with Markdown Headers and Bullet points. Target **~50% line reduction** by removing "weasel words" and flattening structure.

## 3. Pre-Implementation Audit (実装前監査)

- **Concept:** Theoretical defects (e.g., environment incompatibility) must be caught before writing code.
- **Rule:** Always insert a C-3 (Structural Audit) phase before implementation.

## 4. Native Invocation (環境ネイティブ)

- **Concept:** Complex toolchains (snippets, plugins) create friction in chat environments.
- **Rule:** Optimize for "Copy & Paste". Use concise ID shorthand and ensure templates are complete/ready-to-use.

## 5. Portfolio-Ready Default (公開前提)

- **Concept:** Internal tools eventually become public assets.
- **Rule:** Enforce `.gitignore` and `README.md` from Day 1.
